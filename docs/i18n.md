# Internationalization (i18n)

**English** · [简体中文](i18n.zh-CN.md)

## Rules

1. **Primary language is English.** Canonical paths keep the `.md` name (`README.md`, `skills/pro-generation.md`, …).
2. **Chinese is optional and parallel.** Same directory, suffix `.zh-CN.md` (e.g. `README.zh-CN.md`).
3. **Agents MUST read English primary files only.** Do not load `.zh-CN.md` for skill/workflow contracts unless the human explicitly asks for Chinese output.
4. Every bilingual doc SHOULD start with a language switcher, e.g.:

   `**English** · [简体中文](foo.zh-CN.md)`

5. Keep section structure aligned between EN and ZH so diffs stay reviewable.
6. When adding a new doc: ship English first; add `.zh-CN.md` in the same change when practical.

## Naming

| Kind | English (canonical) | Chinese |
|------|---------------------|---------|
| Default | `name.md` | `name.zh-CN.md` |
| Root readme | `README.md` | `README.zh-CN.md` |
| Agent entry | `AGENTS.md` | `AGENTS.zh-CN.md` |

## Maintenance

- Behavior changes go into the **English** skill/workflow first.
- Update Chinese siblings in the same PR when the change is user-facing.
- If ZH lags EN, English wins.
